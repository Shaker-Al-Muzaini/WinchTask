# 🚀 دليل تشغيل نظام إسناد طلبات النقل (Winch Transport System)

هذا الدليل يوضح خطوات تثبيت وتهيئة وتشغيل المشروع بالكامل في البيئة المحلية (Local Environment) للبدء في اختبار خوارزمية التوزيع الجغرافي الذكي ولوحة التحكم اللحظية.

---

## 📋 المتطلبات الأساسية (Prerequisites)
قبل البدء، تأكد من تثبيت البرمجيات التالية على جهازك:
*   **PHP:** إصدار `8.3` أو أحدث.
*   **Node.js:** إصدار `18` أو أحدث (مع مدير الحزم `npm`).
*   **Composer:** لإدارة حزم ومكتبات PHP.
*   **Redis Server:** مثبت ويعمل محلياً على المنفذ الافتراضي `6379`.

---

## 🛠️ خطوات التثبيت والتهيئة (Installation & Setup)

قم بفتح واجهة سطر الأوامر (Terminal) في مجلد المشروع الرئيسي، واتبع الخطوات التالية مرتبة:

### 1️⃣ تثبيت الاعتماديات والمكتبات
```bash
# تثبيت حزم ومكتبات الـ PHP الخلفية
composer install

# تثبيت مكتبات وحزم الواجهة الأمامية (Vue 3 / Inertia)
npm install
```

### 2️⃣ تهيئة ملف الإعدادات البيئية (`.env`)
```bash
# إنشاء ملف الإعدادات من المثال الافتراضي
copy .env.example .env

# توليد مفتاح التشفير الخاص بالتطبيق
php artisan key:generate
```

### 3️⃣ ضبط الاتصال بقاعدة البيانات و Redis
قم بفتح ملف `.env` الجديد وتأكد من ضبط المتغيرات التالية لتتوافق مع بيئتك المحلية:

```env
# قاعدة البيانات (الافتراضية SQLite لسهولة التشغيل الفوري دون إعداد خادم)
DB_CONNECTION=sqlite

# أو يمكنك استخدام خادم MySQL بعد إنشاء قاعدة بيانات باسم wins:
# DB_CONNECTION=mysql
# DB_HOST=127.0.0.1
# DB_PORT=3306
# DB_DATABASE=wins
# DB_USERNAME=root
# DB_PASSWORD=

# إعدادات طوابير العمل والتخزين المؤقت (تتطلب وجود Redis يعمل)
QUEUE_CONNECTION=redis
CACHE_STORE=redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379

# إعدادات خادم بث الويب سوكت اللحظي (Laravel Reverb)
BROADCAST_CONNECTION=reverb
```
> [!IMPORTANT]
> في حال استخدام **SQLite**، تأكد من وجود ملف قاعدة البيانات في المجلد المخصص عن طريق تشغيل الأمر التالي في سطر الأوامر (Windows PowerShell):
> `New-Item database/database.sqlite -ItemType File -Force`
> أو في أنظمة Linux/Git Bash:
> `touch database/database.sqlite`

### 4️⃣ تهجير الجداول وبناء البيانات الافتراضية
```bash
# بناء الجداول وتغذيتها بالبيانات الافتراضية
php artisan migrate:fresh --seed
```

## 🏃‍♂️ تشغيل خوادم التطبيق (Running the App)

لتسهيل تجربة المطور، تم إعداد أوامر تشغيل متزامنة تختصر عليك فتح نوافذ سطر أوامر متعددة.

### 5️⃣ تشغيل خادم التطبيق، Vite، والـ Queue معاً (الأمر الموحد)
قم بتشغيل الأمر السحري التالي في سطر الأوامر:
```bash
composer dev
```
*سيقوم هذا الأمر تلقائياً بتشغيل:*
1.  **خادم Laravel المحلي:** على الرابط `http://127.0.0.1:8000`
2.  **خادم المطورين لـ Vite:** لبناء وتحديث واجهات Vue 3 حياً عند التعديل.
3.  **مستمع طابور المهام (Queue Worker):** لمعالجة خوارزميات الإسناد الجغرافي الخلفية تفادياً لأي بطء.

### 6️⃣ تشغيل خادم البث الحي (Websocket Server)
في نافذة سطر أوامر **جديدة**، قم بتشغيل الأمر التالي لبدء خادم Laravel Reverb لنقل البيانات والتحديثات اللحظية للواجهات:
```bash
php artisan reverb:start
```

## 🎯 تجربة ومحاكاة النظام (Live Simulation)

1.  قم بفتح المتصفح والانتقال للوحة التحكم على الرابط التالي:
    👉 `http://127.0.0.1:8000/index`
2.  ستظهر لك لوحة تحكم تفاعلية أنيقة تعرض **الطلبات النشطة** و **العمليات المسندة**.
3.  لاختبار تدفق البيانات الفوري وخوارزمية التوزيع الجغرافي، يمكنك محاكاة وصول طلبات حية وجديدة في الوقت الفعلي بأكثر من طريقة:
    *   **عبر متصفحك مباشرة:** قم بزيارة الرابط `http://127.0.0.1:8000/api/admin/orders` (ستقوم أداة المحاكاة المدمجة بتوليد طلبات وإحداثيات جغرافية عشوائية وتوزيعها وبثها تلقائياً).
    *   ستلاحظ فوراً ظهور الشحنات الجديدة في لوحة التحكم وتوزيعها الأوتوماتيكي عبر إشعارات حية دون الحاجة لإعادة تحميل الصفحة!
